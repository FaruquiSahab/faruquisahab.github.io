Approach

How I think about engineering

🏗️

Systems over features

I design architectures that absorb the next three features without refactoring. Every product decision starts with data flow, not UI.

🚀

Ship fast, scale smart

Start with the leanest stack that solves the problem. Add complexity when traffic demands it, not when a whiteboard suggests it.

🔒

Security is architecture

Parameterised queries enforced. HMAC-signed documents. JWT with sliding windows. Security isn’t a layer — it’s baked into every function.

📐

Own the full stack

From Figma to Firestore to Cloudflare — I don’t hand off between frontend and backend. One person, one system, zero translation loss.

🧠

Builder-first leadership

I lead a 5-person team by writing code alongside them. Technical leadership means understanding the codebase, not just reviewing PRs.

🌙

2AM engineering mindset

Most ideas start at 2AM and eventually become production features. I enjoy solving problems that only appear at scale.
